利用py2app打包成mac os app 遇到的坑

最近开发一个表情包的项目，需要给产品开发一个把动图拆帧并压缩的工具。代码是用python写的。考虑到产品及运营同学使用方便，就想着把Python脚本封装成一个app, 在网上搜索了半天，最后选择了用py2app 来实现。

教程在网上有很多，就不摘抄了，在这里就简单记录下我记录下py2app 使用的一些坑吧。如果你也踩到了同样的坑，那么恭喜你，下面就是亲测有效的解决办法~

安装

    1. $pip install py2app
    1. cd 项目路径
    1. $py2applet –make-setup helloworld.py

坑1：

  • py2applet: command not found

    解决办法

    • 通过命令查找 py2applet 所在的位置
    • $ sudo find / -name “py2applet” -type f
    • 查找过程比较长，稍微等待一会儿
    • 查找完成，会出现几条path
    • 由于我使用的python2.7, 我选择了下边的地址
    • /System/Library/Frameworks/Python.framework/Versions/2.7/Extras/bin/py2applet
      • 用这个路径再次执行命令
      • /System/Library/Frameworks/Python.framework/Versions/2.7/Extras/bin/py2applet --make-setup helloworld.py
      • 发现显示： Wrote setup.py 就成功了

打包

    1. sudo rm -rf build dist
    1. python setup.py py2app

运行

坑2

  • 双击dist文件夹里的app, 报错: 提示系统权限不足
  • 如何查看出错信息？Contents -> MacOS -> helloworld 可直接运行，弹出终端
  • 代码中需要读写文件，是在当前文件夹中读写
  • 打包成APP之后，读写的文件夹默认是 在 右键-> 显示包内容 -> Contents -> Resources中

  • 所以需要系统权限

  • OSX EI Capitan中增加了一个SIP功能，不管你是不是sudo，都会阻止你在系统目录下进行操作……

  • 如何关闭？

    解决办法

    • 重启电脑，command + R 进入安全模式， 终端输入 csrutil disable

坑3

  • 解决了读写权限问题，发现找不到PIL库

    解决办法

    • 打开setup.py
    • OPTIONS中添加以下内容：
    • OPTIONS = {'argv_emulation': True, 'packages': ['PIL']}

坑4

  • 使用requests 库报错

    解决办法

    • 使用 python自带的urllib库

解决了以上问题， 打包出来的app基本上可以正常运行了!
